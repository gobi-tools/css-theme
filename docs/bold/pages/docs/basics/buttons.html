Two types of buttons styles are supported: standard and outlined (for reset type buttons).

<button>Button</button>
<button type="reset">
  Button
</button>

Both types can be marked as disabled, meaning no interaction will be possible with them.

<button disabled>
  Button
</button>
<button 
  type="reset" 
  disabled>
  Button
</button>

Buttons can be improved by adding svg icons, either to the left or to the right of the main button title.

You can learn more about icons here.

<!-- left side icon -->
<button>
  <svg ...></svg>
  <span>Home</span> 
</button>

<!-- right side icon -->
<button type="reset">
  <span>Play</span>
  <svg ...></svg>
</button>

You can even create icon-only buttons by completely omitting the title.

<button>
  <svg ...></svg>
</button>
<button type="reset">
  <dvg ...></svg>
</button>

You can style buttons semantically with the help of the success and error classes.

You can learn more about classes here.

<button 
  class="success">
  Confirm
</button>

<button 
  type="reset" 
  class="error">
  <svg ...></svg>
  <span>Cancel</span>
</button>

Finally, buttons can be grouped together by wrapping them in a parent tag that has the group class.

You can learn more about groups here.

<p class="group">
  <button>
    Button 1
  </button>
  <button type="reset">
    Button 2
  </button>
  <button type="reset">
    Button 3
  </button>
</p>

<p class="group">
  <button type="reset">
    <svg ...></svg>
  </button>
  <button type="reset">
    <svg ...></svg>
  </button>
  <button>
    <svg ...></svg>
  </button>
</p>

<p class="group">
  <button type="reset">
    Prev
  </button>
  ...
  <button>
    Next
  </button>
</p>
